# Synthetic demonstration assets

Everything in this folder is **synthetic and non-identifiable** — randomly generated by
`../build_demo.py` with a fixed seed. **No PI-CAI data and no patient data of any kind is
used here.** These assets exist only to demonstrate the MoRA console workflow (per-modality
prediction, label-free reliability, reliability-gated fusion, selective deferral, and
modality attribution) without any real data.

- `picai_replay.csv` — synthetic example cases (`DEMO-000` … `DEMO-023`); the demo inputs (committed).
- `*.joblib`, `manifest.json` — lightweight demo models + manifest, **built locally** by
  `build_demo.py` (git-ignored; (re)build any time with `python build_demo.py`).

## Run the demo (from `services/api/`)
```bash
python build_demo.py        # builds the synthetic demo models into this folder
./run_local.sh 8080         # http://127.0.0.1:8080  (admin / changeme)
```
Sign in, pick a `DEMO-###` case, and toggle **break imaging** to watch the imaging
reliability collapse and MoRA down-weight imaging (or defer). The console automatically
uses these assets when no real deployment models are present.
